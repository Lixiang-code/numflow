# Numflow 优化矫正回归报告 v2

> **背景**：上一轮提交 `76220e4`（"全量优化落地"）经实地体检发现实现质量不足——工具造好了但**没有强制使用**，导致测试项目"新侠客闯天涯"中：词表/常数表 0 行、表名中英混拼、坐骑等级硬编码 30 级、所有表 `validation_status=unknown`。本轮在不破坏既有数据的前提下，从代码与提示词层面强制收敛。

## 体检对照（基于 project-b0565c）

| 项 | 上一轮 | 本轮目标 |
|---|---|---|
| `_glossary` 行数 | 0 | 建表自动 upsert |
| `_constants` 行数 | 0 | 公式扫描 → warning 提示 const_register |
| 表名 | `坐骑_属性分配` 中文 | 拒绝中文新建，强制英文 snake_case |
| 坐骑等级 | 写满 200 行但 31 行起值不变；提示词写"坐骑 30 级" | 提示词改为 `system_level_caps[mount]` 派生，禁止硬编码 |
| `validation_status` | unknown | 流水线步骤完成时自动 build_validation_report → passed/warn |

## 本轮代码变更

### 1. 命名层（`backend/app/util/identifiers.py`）
- 新增 `assert_english_ident` / `is_english_ident`，严格 ASCII snake_case（仅 `a-z0-9_`，首字小写字母）
- 旧 `assert_table_or_column` 保留，仅供读路径使用，不破坏旧中文表的兼容读取

### 2. 建表入口（`backend/app/services/table_ops.py`）— 完全重写
- `create_dynamic_table` 强制英文校验；保留中文走 `display_name` / `column_meta[].display_name`
- 建表后**自动**：
  - 调 `_glossary_register_terms` 把表名 + 列名的英中对照 upsert 到 `_glossary`（已有不覆盖）
  - 按 `kind`（默认从表名启发推断：`_alloc/_attr/_quant/_landing/_resource/base_attr`）调 `attach_default_rules` 挂规则
- 新增系统保留名清单（`_glossary/_constants/_validation_history/_agent_sessions/_column_meta`）
- 删除路径仍用宽松 ident（兼容老中文表）

### 3. 校验层（`backend/app/services/validation_report.py`）
- 新增 `default_rules_for(kind, schema_columns, formula_columns)`
- 新增 `attach_default_rules(conn, table, kind, ...)` upsert 到 `_table_registry.validation_rules_json`
- `build_validation_report` 末尾**回写** `validation_status`（`passed` / `warn`），收敛 unknown
- 评估逻辑修复：表无规则 → warn；有规则且无违规 → passed；有违规 → warn
- 硬化 `format_consistency` 对缺失 `_column_meta` 的容错（避免 OperationalError）

### 4. 提示词系统（`backend/app/services/prompt_router.py`）— 整段重写
- 顶部新增 `_NAMING_HEADER`，每段提示词强制贴前缀，含 5 大命名/常数/上限纪律关键字
- **17 段全部用英文表名重写**：`base_attr_table` / `<sys>_alloc` / `<sys>_attr` / `<sys>_cultivation_quant` / `<sys>_landing` 等
- 删除全部魔法数：
  - `@T[等级]` → `@T[level]`
  - HP 起止值 `1000/49000` → `${hp_lv1}/${hp_max_cap}/${growth_exp}`
  - 副本分段值 → `${dungeon_tierN_cap}`
  - 坐骑 30 → `system_level_caps.mount`（"严禁硬编码 30"）
- 每个建表步骤显式要求先 `glossary_register` / `const_register` 再写公式
- fallback 提示词生成器也强制贴前缀

### 5. Agent 自检（`backend/app/services/agent_runner.py:_EXECUTE_SYSTEM_TAIL`）
自诉协议从 3 行扩展为 **6 行**，新增 3 项硬性自检：
- `术语登记`：表/列英中是否已 glossary_register
- `常数登记`：本次公式涉及的字面量是否已 const_register（用 `${name}` 引用）
- `行数依据`：max_level / `system_level_caps[<sys>]` / 品阶枚举 / IFS 分段，**禁止 30/60/100 等魔法数**
命名规则段升级：英文 snake_case 示例 + 反例（拒绝中文表名）

### 6. 流水线钩子（`backend/app/routers/pipeline.py`）
`pipeline_advance` 在快照前自动调 `build_validation_report`，把 `passed/violations_count/warnings_count` 注入返回 payload，迫使下一步上下文携带校验摘要。

### 7. 公式注册常数闸门（`backend/app/services/formula_exec.py`）
`register_formula` 在返回中扫描公式字面量：含小数点或 `>=10` 的整数 → 加 `warnings` 字段提示 `const_register`。`0/1/-1/小整数` 视为索引豁免。

### 8. Agent 工具（`backend/app/services/agent_tools.py`）
- `setup_level_table` 默认 `level_column='level'`，公式样例改为 `${hp_lv1}` 等命名常数
- `create_table` 工具 schema 加 `kind` 参数，dispatch 透传

### 9. 提示词回潮检测（`backend/tests/test_prompts_audit.py`）
新增 5 条断言：
- `_NAMING_HEADER` 含 5 大关键字
- 17 段提示词不含 `_属性分配/_落地/@T[等级]` 等中文 token
- 不含 `坐骑 30 / 30级封顶 / 开放等级 30` 等魔法数
- 建表步骤显式提及 glossary
- 坐骑/翅膀步骤引用 `system_level_caps` 或 `max_level`

`python tests/test_prompts_audit.py` → **5/5 passed**

## 端到端冒烟（Python 直跑，不依赖 LLM）

```
1. create english 'mount_alloc'             → ok
2. create chinese '坐骑_属性分配'            → ValueError 拒绝
3. _glossary auto-registered                → 3 行（表 + 2 列）
4. validation_rules_json non-empty          → 3 条规则（id_not_empty + percent_bounds + format_consistency）
5. build_validation_report 干净数据          → passed=True, status=passed
6. build_validation_report hp_ratio=1.5     → passed=False, violations=1, status=warn
```

## 残留 / 不在本轮范围

- **`regression-fresh-project` E2E**：当前 LLM 为 qwen3.6-flash，提示词更严格后小模型工具调用更易失败；建议用户切到旗舰模型再跑端到端验证。本轮以代码与提示词层面强制约束 + 静态审计 + Python 冒烟覆盖。
- **`placeholder-symbol` / `frontend-display`**：占位符 sigil 选型与前端 display=cn 渲染待用户确认后再做。
- **老项目 project-b0565c 的中文表**：保留可读取，不强制迁移；新代码只拦"新建"。
