# Numflow 优化落地报告（2026-04-26）

> 源：`docs/numflow优化文档.md`
> 计划：`/root/.copilot/session-state/.../plan.md`（37 条 todo 全部完成）

## 一、变更概览

### 1. 数据层（schema + 迁移）

- 新增系统表：
  - `_glossary(term_en, term_zh, kind, brief, scope_table, created_at)`
  - `_glossary_usage(term_en, table_name, column_name)`
  - `_constants(name_en, name_zh, value_json, brief, scope_table, created_at, updated_at)`
- `column_meta` 增加 `display_lang_default`（`cn` / `en` / `raw`）。
- 旧项目自动迁移（`project_migrations.py`）。

### 2. 公式引擎

- 新函数：
  - `CUMSUM_TO_HERE(@T[列])` — 截止本列含本列累计；
  - `CUMSUM_PREV(@T[列])` — 截止上一列累计，第 1 项=0；
  - 通过 `_CURRENT_ROW_INDEX` ContextVar 实现；`eval_series` 检测到 cumsum 强制走逐行路径。
- 公式串支持 `${name}` 常数引用：`execute_formula_on_column` 在求值前调用 `_load_constants` 做数值替换；缺失常数时报清晰错误。
- `register_formula` 新增 `defer=False` 参数；默认注册即计算，杜绝"有式无值"。

### 3. Agent 行为

- `_EXECUTE_SYSTEM_TAIL` 强制注入：
  - **操作前自诉协议**（每次工具调用前必须输出"目标项 / 功能 / 数值设计注意点"三行）；
  - **数值设计原则 6 条**（百分比上下限、性价比拐点、产量小时化、累计公式自洽…）。
- `MAX_EXECUTE_ROUNDS = 24` 硬上限（替代原默认 8）。

### 4. Agent 工具（新增 7 个）

| 工具 | 用途 |
|---|---|
| `glossary_register` / `glossary_lookup` / `glossary_list` | 术语映射表 CRUD |
| `const_register` / `const_set` / `const_list` / `const_delete` | 常数注册 / 修改 / 列出 / 删除（修改触发下游重算） |

### 5. 校验规则（新增 6 条）

- `percent_bounds` — 概率/百分比类列必须在 [0, 0.95]；暴伤放宽至 [0, 10]；
- `format_consistency` — `number_format='0.00%'` 要求存储值在 [0, 1]（拦下整数 350% 写法）；
- `monotone_warning` — 性价比类列禁止严格单调递增（告警，不阻塞）；
- `formula_has_value` — 注册了公式的列任意行为空 → 违规；
- `not_empty_id` — `*_id` / `row_id` / 等级列必须非空；
- `resource_hourly` — 日产量过低 → 建议改为小时产量。

### 6. 流水线结构

- `gameplay_landing_tables` 拆为 7 个子步：`equip / gem / mount / wing / fashion / dungeon / skill`，按项目 `game_systems` 启用列表动态展开。
- 每个子步独立 agent_runner 会话、独立 README、独立 acceptance（`_build_landing_sub_spec`）。
- `pipeline_state` 兼容旧数据：`completed_steps` 中含 `gameplay_landing_tables` 自动视为所有子步均 done。
- 前端 `pipelineSteps.ts` 增加子步标签 + 智能 fallback。

### 7. 提示词路由

- `prompt_router.DEFAULT_STEP_PROMPTS` 重写：
  - step_2 / 4 / 6 / 9 / 10：硬约束"行数 == max_level"，给出 `IFS(@T[等级]<=10, 30, ...)` 范式；
  - step_7 / 8 / 9：产量列改为 `hourly_yield_*`，普通资源量大、高级资源量小；
  - 副本表必含「累计门票 / 性价比 / 通关门槛」三列（cumsum + IFS）；
- 7 个 per-system landing 子步独立提示词，约束本子步只产出本系统表。

### 8. 数据 API

- `GET /data/tables/{t}/rows?display=cn|en|raw`（默认 cn）：按 glossary 翻译列名/列值。
- `GET /data/tables/{t}` 返回 `related_constants[]`（表级 + 全局）。

### 9. 前端

- `Workbench.tsx`：右侧面板新增「相关常数」详情卡片（与快照、校验同级），读取 `related_constants`。
- `pipelineSteps.ts`：子步标签 + fallback。
- `npm run lint`：无新增错误（仅 `ProjectSetup.tsx` 既有问题，与本次改动无关）；
- `npx tsc --noEmit`：✅ pass；
- `npm run build`：✅ 成功输出 `dist/`。
