# 表名/列中文化、分玩法分表、公式在客户端的展示

**更新日期**：2026-04-24  

## 1. 表名与列名（中文优先）

- 动态业务表、列名在服务端统一由 [backend/app/util/identifiers.py](file:///www/wwwroot/numflow/backend/app/util/identifiers.py) 校验：允许 **中文、英文、数字、下划线**；首字符不能为数字，且不得以 `_` 开头（与系统表 `_` 前缀区分）。
- 创建表、写入单元格、公式中的 `@表名[列名]` 引用均须符合同一套标识规则；公式引擎 [formula_engine.py](file:///www/wwwroot/numflow/backend/app/services/formula_engine.py) 已支持 **CJK 表/列** 的引用解析。
- **发版后请重启 Uvicorn**，使新规则对线上进程生效。

## 2. 分玩法：一表一系统（二级表）

- 「一阶/二阶」为**全游戏共用框架**；**各玩法系统**的二级数值（分玩法贡献、分玩法资源可用、分玩法落地）在数据模型上**单独成表**（例如 `装备系统_玩法属性`、`装备系统_资源可用`、`装备系统_落地`），与「多系统混在一张表」的写法脱钩，便于按系统浏览与维护。
- 实机 E2E 建表与命名约定见 [scripts/real_flow_e2e.py](file:///www/wwwroot/numflow/scripts/real_flow_e2e.py)（全中文表头 + 分表）。

## 3. 公式在客户端的可见性

- `GET /api/data/tables/{表名}` 的返回体增加 **`column_formulas`**：形如 `{ "列名": "已注册公式字符串" }`（与 `_formula_registry` 一致）。
- 工作台 [Workbench.tsx](file:///www/wwwroot/numflow/frontend/src/pages/Workbench.tsx) 中，对**存在公式的列**在表头展示：
  - **`title` 悬停**（浏览器原生）为 `公式：…`；
  - 样式上增加 **ƒ 角标** 与**虚线底边**（`col-has-formula`），避免用户误以为与纯手填列相同。
- 一阶表 E2E 在列 **`防御验算_公式`** 上注册了示例 `0.5*@一阶属性框架[攻击力]` 并执行，便于在界面验证。

## 4. AGENT TEST（独立监控与导出）

- 路由 **`/agent-test`**（亦可在工作台顶栏点 **「AGENT TEST」** 进入，并带当前 `?project=`）。
- 与 Workbench 使用**同一** `POST /api/agent/chat`（含 `X-Project-Id`、Cookie），**完整记录** SSE 每条 `data:` 事件（时间戳 + 原始 JSON）。
- 提供 **工具调用表**（`tool_call` / `tool_result` 行）与 **全量事件区**；支持 **导出 JSON**（含 `exportVersion`、session 元数据、events 数组）与 **NDJSON**（每行一事件，含 `_ts`）。
- 最近一次会话会尝试写入 `localStorage`（过大时可能失败），可用「从本机恢复上次」加载。

## 5. 表列表的辅助信息

- 表清单除 `table_name` 外，在左侧对 **`purpose`（建表时写入的用途/流水线步）** 做第二行小字显示，帮助在中文长表名下仍能对齐流水线语义。

## 6. 无用测试项目清理

- 脚本 [scripts/cleanup_test_projects.py](file:///www/wwwroot/numflow/scripts/cleanup_test_projects.py)：删除 `server.sqlite` 中符合条件的项目行，并删除 `data/projects/{slug}/` 目录。
- 匹配规则见脚本内 `is_test_row`（如 `03e2e-`、`e2e-` 等）；**`NUMFLOW_CLEANUP_DRY=1` 为仅打印。**
- **不删除** `slug=template` 的模板项目。
