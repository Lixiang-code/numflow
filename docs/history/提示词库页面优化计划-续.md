## 二、设计评价（全轮）

### 优点

1. **信息层级清晰**  
   `sl-input-ai`（绿色实线）vs `sl-input-dev`（灰底虚线）给"AI 可见 / 仅开发者"建立了视觉语义，认知成本低。

2. **预览内联化符合操作心智**  
   预览从顶部固定改为内联在"基本信息 → 预览 → 模块"之间，建立了"我改模块 → 看到输出"的空间因果关系；比顶部固定预览区语义更强。

3. **AutoTextarea 体验接近原生**  
   默认行数 = 内容行数 + 1、超 10 行启用滚动条（单模块 30 行），比固定高度输入框舒适，且不影响布局稳定性。

4. **草稿缓存设计合理**  
   `useRef(Map)` 避免每次击键触发全量渲染；`useState(Set)` 脏状态保证 UI 响应式更新；两者职责分离。

5. **模块级精准[编辑中]标记**  
   从"整条条目编辑中"精细化到"具体哪个模块编辑中"，让多模块场景下的改动范围一目了然。

6. **侧边栏启用/未启用分组**  
   开发者可立即识别哪些提示词是生效状态，管理效率提升。

7. **页面级控制栏设计正确**  
   [启用][默认暴露] 是全局开关，始终可见优于"收起时才看到"，且横向排布降低了控件占用的纵向空间。

### 不足 / 隐患

1. **预览区内联后违背了"不被滚动挪走"的原始需求**  
   之前置顶是 P0 需求，内联化后当页面内容多时预览会随滚动消失。建议给预览区加 `position: sticky`，让它"跟随但不消失"。

2. **页面控制栏缺少脏状态反馈**  
   修改启用/暴露状态后，sl-page-controls 本身没有任何视觉变化提示"有未保存的改动"（基本信息区的 sl-section-dirty 覆盖不到这里）。

3. **`deepEqual = JSON.stringify` 的脆弱性**  
   key 顺序不同时会误报"不等"，一旦后端返回字段顺序变化就产生假冲突。应替换为 `fast-deep-equal`。

4. **"稍后再说"后侧边栏无冲突视觉提示**  
   冲突弹窗关闭后，该条目看起来和正常条目一样，用户容易遗忘。

5. **新建 SKILL（id='new'）的草稿在刷新后丢失**  
   缓存在内存中，刷新即失，无警告。

6. **Ctrl+S 在 canSave=false 时静默失败**  
   没有提示用户"为什么不能保存"。

7. **侧边栏"未启用"分组不可折叠**  
   当未启用条目很多时，启用中的条目会被推至很远，影响日常管理效率。

8. **缓存没有大小上限**  
   Map 无 LRU/TTL，长期 SPA 使用可能积累孤立条目。

## 三、同一思路链条下尚未执行、但应当执行的内容

### 优先级 P0（修复现有体验缺陷）

#### 3.1 预览区 sticky 化
- 给 `.sl-preview-inline` 加 `position: sticky; top: 0; z-index: 10`（需确认滚动容器是 `.sl-main`）
- 这样预览跟随滚动容器顶部固定，不随内容区一起滚走，同时保持内联语义位置

#### 3.2 页面控制栏脏状态标记
- `skillBasicDirty`（或新增 `skillControlsDirty`）为 true 时，在 `.sl-page-controls` 上加一个小橙点或边框
- 样式建议：`border-bottom: 2px solid #f59e0b`，和模块脏状态保持一致的色调

#### 3.3 "稍后再说"后侧边栏冲突视觉提示
- 弹窗关闭后在侧边栏该条目上加 `⚡` 图标（amber 色），hover 提示"存在未解决冲突"
- 下次 selectSkill 时仍会触发弹窗（serverConflict 未被删除，已验证 ✅）

#### 3.4 Ctrl+S 失败时提示原因
- `handleSave` 中 `canSave=false` 时调用 `showToast('error', '请先填写标题才能保存')`

### 优先级 P1（体验提升）

#### 3.5 侧边栏"未启用"分组可折叠
- 点击分组标题可折叠/展开该组，折叠状态存 `localStorage`
- 折叠后仅显示 `未启用 (N)` 计数

#### 3.6 新建 SKILL 草稿 localStorage 持久化
- 仅对 id='new' 的草稿写入 `localStorage`（key: `sl_draft_new`）
- 加载页面时检查并恢复；离开页面前 `beforeunload` 警告

#### 3.7 离开页面前警告（beforeunload）
- 有 dirtySkillIds 或 dirtyPromptKeys 时注册 beforeunload 事件

#### 3.8 模块批量启用/禁用
- 模块区顶部加"全部启用 / 全部禁用"快捷操作

### 优先级 P2（工程质量与扩展性）

#### 3.9 替换 deepEqual 实现
- `npm install fast-deep-equal`，替换 `JSON.stringify` 比较

#### 3.10 缓存 LRU 限制
- 给 skillCache / promptCache 加最大容量（50 条），超出删除最久未访问条目

#### 3.11 工具提示词路径可视化
- 在每个工具模块标题旁展示路径面包屑（`function → parameters → sql → description`）

#### 3.12 提示词字符/token 估算
- 在预览区下方展示"约 N token"（字符数/3.5 估算），帮助控制提示词长度

### 缓存结构
```typescript
// key = String(skillId) 或 'new'
skillCache.current: Map<string, SkillCacheEntry>
// key = `${tab}::${prompt_key}`
promptCache.current: Map<string, PromptCacheEntry>

type SkillCacheEntry = {
  draft:           SkillItem   // 当前编辑内容
  baseline:        SkillItem   // 上次从服务端加载的快照
  serverConflict?: SkillItem   // 服务端更新但本地有草稿时暂存
}
```
