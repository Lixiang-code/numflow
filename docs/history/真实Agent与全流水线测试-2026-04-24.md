**依据**：[游戏数值系统AI化自动开发-00-引导与索引.md](../游戏数值系统AI化自动开发-00-引导与索引.md)、[游戏数值系统AI化自动开发-03-数值模型首次构建流程.md](../游戏数值系统AI化自动开发-03-数值模型首次构建流程.md)（**必须按章节有表/文档级产出，而非单表两三行**）、[全链路实机测试报告-2026-04-24.md](./全链路实机测试报告-2026-04-24.md) 与 [scripts/real_flow_e2e.py](../../scripts/real_flow_e2e.py)。

**说明**：早期一版 E2E 仅建 `num_base_framework` 两示例行，**不符合** `03` 对「全等级一阶表、时间/经验、方案、分配、二阶、分玩法、养成、落地」的宽度要求；当前脚本已改为**分流水线步建表、全等级（默认 60 级）填数、约 12 张业务表、数千单元格写入**。

---

## 1. 与 `03` 章节的表级映射（摘要）

| 流水线 step / `03` 概念 | 本脚本落库（示例名） | 行数/规模（60 级、4 玩法时） |
|-------------------------|----------------------|------------------------------|
| 环境 + 全局 README | `PUT /meta/global-readme` + `e2e_pipeline_index` | 11 行索引 + 长文 README |
| 一阶 + 时间/经验 | `num_base_framework` + `num_level_pacing` | 各 **60 行**；含暴击/对抗/双技能线、停留与经验目标 |
| 玩法属性方案 | `num_gameplay_attr_scheme` | 每玩法 1 行方案说明 |
| 分配%（可不求和=100%） | `num_alloc_const` | 4×5 单元 |
| 二阶框架 | `num_second_order` | **60 行**（一阶按分配合成） |
| 分玩法具体属性 | `num_gameplay_by_system` | **4×60=240 行** |
| 养成资源设计 | `cult_res_catalog` | ≥2 基础货币 + 专用品，多行 |
| 资源分等级产出 | `num_res_per_level` | 4 资源×**60 级** |
| 资源→玩法权重 | `num_cult_to_system` | 多行，权重和可≠1 |
| 分玩法可用量 | `num_cult_available` | 4×**60** |
| 落地参考 | `num_gameplay_landing` | 4×**60**（消耗/掉率权重等） |

**自动化输出指标（`NUMFLOW_DATA_ONLY=1` 一次实跑）**：`table_count_dynamic ≈ 12`，`cell_writes_estimated` 约 **5.5k+**，抽样 API 行计数约 **1.1k+ 行**（`GET /data/.../rows?limit=5000`，与后端 `limit` 上限一致）。

**环境变量**

- `NUMFLOW_LEVEL_MAX`：等级行数（默认 60，与 `settings.core.level_max` 一致）
- `NUMFLOW_DATA_ONLY=1`：跳过千问，仅跑建表+流水线+HTTP 检，用于无密钥/快速回归

## 2. 用例 A：按步建表后 `advance`（与「空点流水线」区别）

- 对 `PIPELINE_STEPS` 每一步先 `setup_step_tables`（该步**应有**的表+README+数据），再 `POST /api/pipeline/advance`。
- 首步**错序**、结束后**多 advance** 仍 400、快照 `pipeline:…` 链、验证与 `echo_sum` 等逻辑与 2026-04-24 全链路报告一致。

## 3. 用例 B / C：Agent、断流、幂等

- 默认（非 `DATA_ONLY`）下：Agent 指令改为在**大表**上改 **L30.hp** 等、并 `update_table_readme` 至 **`num_gameplay_landing`**，与「只改 2 行小表」脱钩；仍保留 SSE 半连接与幂等 `write`。
- **判负**：仅一张表、个位数行/单元格、却声称跑完「`03` 等价的真实测试」。

**总评**：当前自动化用「**多表 × 全等级行 × 分配/养成尺度**」对齐 [03](./游戏数值系统AI化自动开发-03-数值模型首次构建流程.md) 的**宽度**；深度（公式迭代、全表 @ 引用、各玩法子表无限细化）仍可在该骨架上由策划/Agent 继续加表加列，而非再退回「单表两行」的烟测假阳性。
