# 前端 Lint 修复日志

> 日期：2026-04-30  
> 范围：`frontend/src/pages/*.tsx`

## 背景

此前 `frontend` 全量执行 `npm run lint` 时，存在一批既有问题，主要集中在 React Hooks 新规则上，包括：

1. `react-hooks/set-state-in-effect`
2. `react-hooks/refs`
3. `react-hooks/purity`
4. `@typescript-eslint/no-unused-vars`
5. `react-hooks/exhaustive-deps`（warning）

这些问题虽然不一定立即导致线上故障，但会让前端代码在状态同步、渲染稳定性和后续维护上存在隐患。

## 本次修复范围

### 1. MatrixEditor / ThreeDimTableEditor

- 去掉在 `useEffect` 内同步调用 `setLoading(...)`、`setFixedValue(...)` 的写法
- 改为使用请求键或派生值来表达“当前是否仍在加载 / 当前固定维度有效值”
- 避免 effect 触发级联渲染

### 2. NewProject

- 去掉 `Date.now()` 生成自定义子系统 ID 的写法
- 改为基于路径和已有数量生成稳定序号 ID
- 避免触发 `react-hooks/purity`

### 3. ProjectSetup

- 去掉“根据 `live` 变化再同步 `setOpen(...)`”的 effect
- 改为“手动展开态 + 派生默认展开态”的模式
- 去掉未使用参数
- 将初始化加载改为异步调度，避开 `set-state-in-effect`

### 4. SkillLibrary

- 清理渲染阶段对 `ref.current` 的直接/间接读取
- 为缓存补充可渲染的 state 镜像，避免 render 闭包中直接触碰 ref
- 让草稿、baseline、冲突信息在渲染层有稳定快照

### 5. Workbench

- 修复 cleanup 中直接引用 ref.current 带来的 lint warning
- 在 effect 内先捕获局部引用，再在卸载时清理

## 结果

- `frontend` 全量 `eslint` 已通过
- `frontend` 构建已通过
- 这次修改没有扩大到无关业务逻辑，重点只处理了 lint 报出的既有问题
